# טראק Claude Code Mastery — עיצוב מלא

מטרה: **המשאב המקיף והמעשי ביותר ללימוד Claude Code** — ממתחיל מוחלט ועד רמת מהנדס AI בכיר
שעובד בפרודקשן. הטראק בנוי סביב תהליכי עבודה אמיתיים (workflows), לא דוגמאות מבודדות, ומורחב
בכל שיעור ע"י שיפור אמיתי ב-AtlasDesk (הקפסטון המתמשך של האקדמיה).

עקרונות עיצוב:
- כל שיעור: חשיבה **לפני** קוד (תכנון/ארכיטקטורה/פירוק) → ואז ביצוע עם Claude Code.
- השוואות זה-לצד-זה: פרומפט טוב/גרוע, workflow יעיל/לא יעיל, context קטן/גדול, ידני/AI-assisted,
  תיקון-מהיר/ארכיטקטורה-ארוכת-טווח — רכיב `PromptComparisonLab` חדש לשימוש חוזר בכל השיעורים.
- כל שיעור מסתיים ב**משימה מעשית אמיתית** (`RealWorldTask`) — התלמיד פותח את Claude Code שלו על
  מחשבו ומבצע משימת הנדסה אמיתית (על AtlasDesk עצמו, שהקוד שלו ציבורי בריפו של האקדמיה), לא רק קורא.
- כל מודול מסתיים בפרויקט שמרחיב את AtlasDesk בפועל (יכולת נראית/נבדקת), מתועד ב-
  [13-atlasdesk-features.md](13-atlasdesk-features.md).

## מבנה: 6 מודולים, ~26 שיעורים

### מודול 1 — יסודות ותשתית (Foundations & Setup)
1. **התקנה והגדרה** — installation, auth, `claude` CLI, IDE integrations, הגדרות גלובליות מול פרויקט
2. **מבנה פרויקט ו-CLAUDE.md** — מה שם בקובץ הזיכרון, מוסכמות, מבנה תיקיות שעוזר ל-AI להבין קוד
3. **יסודות ניהול Context** — context window בפועל בסשן פיתוח, מתי לנקות/לאתחל, `/compact`, קבצים גדולים מול ממוקדים
4. **פרויקט מודול: קליטת AtlasDesk ל-Claude Code** — בניית CLAUDE.md אמיתי לפרויקט, סבב שאלות תכנון ראשון

### מודול 2 — חשיבה לפני קוד: תכנון וארכיטקטורה
1. **עיצוב פרומפטים לקוד** — good vs poor prompts (השוואה חיה), specificity, דוגמאות/anti-דוגמאות
2. **תהליכי תכנון (Planning Workflows)** — plan-mode, מסמכי תכנון לפני ביצוע, אישור תוכנית לפני קוד
3. **פיתוח Architecture-First** — לתכנן מבנה/ממשקים לפני מימוש, למה AI בלי ארכיטקטורה מייצר בלגן
4. **פירוק משימות גדולות (Decomposition)** — לפרק feature גדול לצעדים קטנים ניתנים לאימות
5. **פרויקט מודול: תוכנית ארכיטקטורה ל-AtlasDesk feature אמיתי + מימוש**

### מודול 3 — תהליכי פיתוח ליבה (Core Development Workflows)
1. **בניית פיצ'רים מאפס** — workflow מלא: תכנון→שלד→מימוש→בדיקה
2. **תהליכי Refactoring בטוחים** — refactor מודרך, שמירה על התנהגות, בדיקות לפני/אחרי
3. **תהליכי Debugging** — לתת ל-Claude Code הקשר שגיאה נכון, hypothesis-driven debugging
4. **TDD עם Claude Code** — לכתוב טסטים קודם, לתת ל-AI לממש עד שהם עוברים
5. **פרויקט מודול: פיצ'ר אמיתי ב-AtlasDesk בגישת TDD מלאה**

### מודול 4 — עבודה עם קוד קיים ופרויקטים גדולים
1. **עבודה עם קוד בסיס קיים** — לגרום ל-AI "להכיר" קוד זר לפני שינוי
2. **פיתוח פיצ'רים אינקרמנטלי** — שינויים קטנים ובטוחים במקום מהפכות
3. **ארגון פרויקטים גדולים** — מודולריות, monorepo-style, מוסכמות שמסייעות ל-AI לנווט
4. **תהליכי Code Review** — לבקש מ-Claude Code לבקר קוד (גם קוד שהוא עצמו כתב), checklist ביקורת
5. **פרויקט מודול: הרחבה משמעותית של AtlasDesk תוך שמירה על ארכיטקטורה קיימת**

### מודול 5 — משמעת הנדסית ופרודוקטיביות
1. **תהליכי Git עם Claude Code** — commits משמעותיים, הודעות קומיט, ניהול branches, PR flow
2. **תהליכי תיעוד** — לגרום ל-AI לתעד קוד/החלטות אדריכליות תוך כדי עבודה, לא בדיעבד
3. **ניהול סשן ושימור Context** — סשנים ארוכי-טווח, מתי לפצל, `/resume`, שימור החלטות בין סשנים
4. **ספריות פרומפטים ותבניות לשימוש חוזר** — לבנות "playbook" אישי של פרומפטים שחוזרים
5. **פרויקט מודול: prompt-library מתועד ב-repo של AtlasDesk + שיפור workflow אמיתי**

### מודול 6 — הנדסת Production מתקדמת
1. **אופטימיזציית ביצועים** — לזהות ולתקן צווארי בקבוק בעזרת Claude Code
2. **התאוששות משגיאות וסשנים ארוכים** — מה עושים כש-Claude Code "נתקע"/סוטה, checkpoint strategy
3. **תהליכי דיפלוי לפרודקשן** — לשלב Claude Code בצינור build→test→deploy→verify
4. **טעויות נפוצות ואנטי-פטרנים** — trust-blindly, קונטקסט חסר, אין בדיקות, prompt מעורפל
5. **טכניקות הנדסיות מתקדמות** — sub-agents, מקבילי workflows, self-review loops
6. **קפסטון: AtlasDesk מגיע לרמת production-ready אמיתית** — הכל ביחד: ביצועים+יציבות+תיעוד+דיפלוי
